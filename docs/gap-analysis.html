⚙️ ARCHITECTURAL DIFFERENCES Quality & Performance
Testing Coverage
Kotlin: 48 test files with comprehensive coverage of SMS parsing, navigation, repositories, and financial calculations.
Flutter: 29 test files with solid coverage but may lack edge cases for SMS dedup and financial logic.
Gap: Flutter should expand test suite for SMS parser, financial calculations, and edge cases.
Database Encryption Implementation
Kotlin: Room with SQLCipher, PBKDF2 password hashing, optional encryption for new installs.
Flutter: Drift with SQLCipher and crypto package for hashing.
Gap: Ensure Flutter's encryption strategy matches Kotlin's PBKDF2 implementation exactly.
Background Job Scheduling
Kotlin: WorkManager with multiple specialized workers (SMS sync, recurring execution, daily digest, audit cleanup).
Flutter: Workmanager with coordinator pattern, but may lack job variety and sophistication.
Gap: Flutter should implement specialized workers for each background task type (SMS, recurring, daily digest, cleanup).
Navigation Architecture
Kotlin: Jetpack Navigation Compose with type-safe routes, deep linking, bottom nav + secondary routes.
Flutter: GoRouter with similar capabilities.
Gap: Verify Flutter's deep linking implementation matches Kotlin's coverage and test with same scenarios.
State Management Patterns
Kotlin: MVVM with ViewModel + StateFlow + sealed event classes.
Flutter: Riverpod with AsyncNotifier and StateNotifier.
Gap: Both are solid; ensure Flutter's Riverpod patterns mirror Kotlin's unidirectional data flow.
Sync & Conflict Resolution
Kotlin: Explicit sync metadata (LOCAL_ONLY, QUEUED, SYNCING, SYNCED, FAILED, CONFLICT, TOMBSTONED) with conflict resolution via user-scoped keys.
Flutter: Has sync support but may lack Kotlin's conflict resolution sophistication.
Gap: Flutter should implement identical sync state machine and conflict detection.

💡 IMPLEMENTATION NOTES & QUICK WINS Best Practices
SMS Parsing Improvements (Quick Win)
Kotlin's confidence scoring uses regex pattern matching with explicit confidence levels. Implement identical scoring in Flutter's SMS parser to match Kotlin's behavior exactly. Test with the same SMS fixtures used in Kotlin's parser tests.
Financial Dashboard Visualization
Use fl_chart (already in pubspec) to implement spending velocity charts matching Kotlin's FinanceScreen. Include weekly trend arrows and pie charts for category breakdown. Consider reusing the same data models and calculations from Kotlin.
Transaction Categorization Workflow
This is a critical blocker for many downstream features. Implement as a BottomSheet or Page with:
  • Category selector with icons and colors
  • Transaction preview
  • Bulk categorization option
  • Merchant learning feedback loop
Test Coverage Gap
Flutter currently has 29 test files vs Kotlin's 48. Prioritize tests for:
  • SMS parser edge cases (confidence scoring, dedup)
  • Financial calculations (budget progress, spending velocity)
  • Recurring rule execution and date logic
  • Sync conflict resolution
Database Schema Alignment
Ensure Drift tables match Room entities exactly, including:
  • sync_state enum (LOCAL_ONLY, QUEUED, SYNCING, SYNCED, FAILED, CONFLICT, TOMBSTONED)
  • source_hash and semantic_hash for SMS dedup
  • confidence_score for SMS parse quality
  • quarantine_reason for low-confidence entries
Background Job Architecture
Kotlin has 4 distinct WorkManager jobs. Flutter should implement equivalent workmanager tasks:
  • SMS Sync Worker: Periodic historical SMS scan (hourly)
  • Recurring Execution Worker: Hourly materialization of due recurring rules
  • Daily Digest Worker: Daily summary notifications
  • Import Audit Cleanup: Prune old audit logs weekly
UI/UX Component Parity
Both apps use glassmorphism, but ensure Flutter has equivalent components:
  • MetricCard — with trend arrows (↑↓)
  • BudgetProgressIndicator — circular progress with category color
  • ImportHealthPanel — import quality visualization
  • SegmentedControl — for period/view switching
